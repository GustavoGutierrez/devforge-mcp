MCP Integration Guide

devforge-mcp exposes tools via stdio transport only. This keeps deployment simple, local-first, and secure by default.

Client configuration example

{
  "mcpServers": {
    "devforge": {
      "command": "/path/to/devforge-mcp",
      "args": [],
      "env": {}
    }
  }
}

Stateless execution model

  • No database writes or persistent runtime state between calls.
  • Deterministic utility behavior for reproducible agent outputs.
  • Structured JSON responses and explicit error objects.

How agents should use DevForge tools

  1. Choose the minimal tool that solves the task.
  2. Pass explicit parameters (avoid ambiguous defaults when precision matters).
  3. Chain outputs into next steps only when required.
  4. Treat file-writing tools carefully in production repos.

Tool naming

Canonical MCP names in this server are unprefixed (for example markdown_to_pdf). Some clients display prefixed aliases such as devforge_markdown_to_pdf. Behavior and arguments are equivalent.
