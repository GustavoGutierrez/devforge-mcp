CLI / TUI Usage

The devforge binary exposes the same tool surface available through MCP, but optimized for interactive terminal workflows.

Why use the CLI

  • Manual execution and debugging of tool calls before wiring into agents.
  • Local productivity for one-off conversions, formatting, cryptography, and media processing.
  • Fast validation of arguments and output structure.

Basic commands

devforge --help
devforge

Typical workflow

  1. Pick a tool (for example markdown_to_pdf).
  2. Provide required and optional inputs.
  3. Inspect structured JSON response.
  4. Reuse the same payload in MCP-enabled agents.

CLI + MCP parity

Tool names, argument shapes, and behavior should remain consistent between CLI and MCP server.
